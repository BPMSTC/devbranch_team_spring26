# Jira Backlog Documentation Pack

This folder contains a complete, professional set of documents to define, refine, prioritize, and execute your Jira backlog consistently.

## Contents

1. `01-backlog-governance.md`
   - Backlog ownership, operating cadence, quality gates, and review workflow.
2. `02-epic-template.md`
   - Standardized format for epics with business value, scope, and dependencies.
3. `03-user-story-template.md`
   - Ready-to-copy story format with acceptance criteria and technical notes.
4. `04-prioritization-model.md`
   - Practical scoring framework for ranking backlog items.
5. `05-sprint-planning-and-capacity.md`
   - Sprint planning workflow, capacity model, and commitment guardrails.
6. `06-jira-workflow-and-statuses.md`
   - Suggested Jira statuses, transitions, and ticket hygiene rules.
7. `07-release-readiness-checklist.md`
   - Final quality and delivery checklist before release.

## How To Use

- Start with `01-backlog-governance.md` and align your team on standards.
- Create epics using `02-epic-template.md`.
- Break epics into stories using `03-user-story-template.md`.
- Rank items with `04-prioritization-model.md`.
- Plan each sprint with `05-sprint-planning-and-capacity.md`.
- Configure Jira board/workflow based on `06-jira-workflow-and-statuses.md`.
- Run release checks using `07-release-readiness-checklist.md`.

## Suggested Jira Custom Fields

- Business Value (1-5)
- Time Criticality (1-5)
- Risk Reduction / Opportunity Enablement (1-5)
- Job Size (1-13)
- Dependencies (text or issue links)
- Target Release (version)
- Definition of Ready (checkbox)
- Definition of Done (checkbox)
