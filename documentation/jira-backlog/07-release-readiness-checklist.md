# Release Readiness Checklist

Use this before promoting sprint output to release.

## Scope and Traceability

- [ ] Release scope is finalized and approved.
- [ ] All included Jira items are in `Done`.
- [ ] Every change maps to a tracked ticket.

## Quality

- [ ] Acceptance criteria verified for all included stories.
- [ ] Regression suite passed.
- [ ] Critical/high bugs resolved or explicitly accepted.
- [ ] Performance and accessibility checks completed (as applicable).

## Security and Compliance

- [ ] Security-impacting changes reviewed.
- [ ] Required compliance checks completed.
- [ ] Secrets/config changes validated in target environment.

## Deployment Readiness

- [ ] Deployment steps documented and tested.
- [ ] Rollback plan documented and validated.
- [ ] Monitoring/alerts are in place for new behavior.

## Communication

- [ ] Release notes prepared.
- [ ] Stakeholders notified of scope and timing.
- [ ] Support/on-call aware of release window.
